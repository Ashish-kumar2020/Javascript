# 📘 JavaScript `this` Keyword

---

# What is `this`?

`this` is a special keyword that refers to the object that is currently calling or executing the function.

For **normal functions**, the value of `this` is determined by **how the function is called**, not where it is defined.

---

# Global `this`

```javascript
console.log(this);
```

### Browser

```text
Window Object
```

---

# `this` Inside an Object Method

```javascript
const person = {
    name: "Ashu",

    greet() {
        console.log(this.name);
    }
};

person.greet();
```

Output

```text
Ashu
```

---

## Why?

```javascript
person.greet();
```

The caller is

```text
person
```

So

```text
this = person
```

Internally

```javascript
console.log(person.name);
```

---

# `this` Refers to the Entire Object

```javascript
const person = {
    name: "Ashu",

    greet() {
        console.log(this);
    }
};

person.greet();
```

Output

```javascript
{
    name: "Ashu",
    greet() {}
}
```

`this` refers to the complete object, not just one property.

---

# Losing `this`

```javascript
const person = {
    name: "Ashu",

    greet() {
        console.log(this.name);
    }
};

const fn = person.greet;

fn();
```

Output (Browser)

```text
undefined
```

---

## Why?

```javascript
fn();
```

There is **no object** calling the function.

So in the browser (non-strict mode):

```text
this = window
```

Internally

```javascript
console.log(window.name);
```

Since `window.name` is not the `name` property from `person`, the result is typically `undefined` in interview examples.

> **Note:** In strict mode or ES modules, `this` inside a standalone function is `undefined`.

---

# Changing the Caller

```javascript
const obj1 = {
    name: "Ashu"
};

const obj2 = {
    name: "John",

    greet() {
        console.log(this.name);
    }
};

obj1.greet = obj2.greet;

obj1.greet();
```

Output

```text
Ashu
```

---

## Why?

Although the function originally belonged to `obj2`, it is called as

```javascript
obj1.greet();
```

The caller is

```text
obj1
```

So

```text
this = obj1
```

---

# Returning `this`

```javascript
const person = {
    name: "Ashu",

    greet() {
        return this;
    }
};

console.log(person.greet());
```

Output

```javascript
{
    name: "Ashu",
    greet() {}
}
```

---

# Golden Rule ⭐⭐⭐⭐⭐

For **normal functions**:

```text
Ignore where the function was created.

Ignore where the function was assigned.

Only ask:

Who is calling the function?

That object becomes `this`.
```

# Interview Questions

## What is `this`?

`this` is a special keyword that refers to the object currently calling or executing the function.

---

## Does `this` depend on where a function is defined?

No.

For normal functions, `this` depends on **how the function is called**.

---

## What is the value of `this` in

```javascript
person.greet();
```

`this` refers to `person`.

---

## Why does this print `undefined`?

```javascript
const fn = person.greet;

fn();
```

Because there is no object calling the function.

In a browser (non-strict mode), `this` becomes the global object (`window`), so `this.name` is not the `person.name` value.

---

## Can the same function have different values of `this`?

Yes.

```javascript
obj1.greet = obj2.greet;

obj1.greet();
```

`this` becomes `obj1` because `obj1` is the caller.
