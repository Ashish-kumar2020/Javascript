# 📘 JavaScript `bind()`

---

# What is `bind()`?

`bind()` is a method that creates and returns a **new function** with the value of `this` permanently set.

Unlike `call()` and `apply()`, `bind()` **does not execute the function immediately**.

---

# Syntax

```javascript
const newFunction = functionName.bind(thisArg, arg1, arg2);
```

### Parameters

- **thisArg** → Object that should become `this`
- **arg1, arg2...** → Arguments passed individually (optional)

---

# Basic Example

```javascript
function greet() {
    console.log(this.name);
}

const user = {
    name: "Ashu"
};

const fn = greet.bind(user);

fn();
```

Output

```text
Ashu
```

---

# Why?

```javascript
const fn = greet.bind(user);
```

does **not** execute `greet()`.

It creates a **new function** that remembers

```text
this = user
```

When we later call

```javascript
fn();
```

the function executes with

```text
this = user
```

---

# `bind()` Does Not Execute Immediately

```javascript
function greet() {
    console.log(this.name);
}

const user = {
    name: "Ashu"
};

const fn = greet.bind(user);

console.log(fn);
```

Output

```text
ƒ bound greet() { ... }
```

or

```text
[Function: bound greet]
```

because `bind()` returns a new function instead of executing it.

---

# `bind()` with Arguments

```javascript
function greet(city) {
    console.log(this.name);
    console.log(city);
}

const user = {
    name: "Ashu"
};

const fn = greet.bind(user, "Delhi");

fn();
```

Output

```text
Ashu
Delhi
```

---

# What Happens Internally?

```javascript
const fn = greet.bind(user, "Delhi");
```

Think of it like this:

```javascript
function fn() {
    greet.call(user, "Delhi");
}
```

`bind()` creates a new function that remembers:

- `this`
- Arguments (if provided)

The function runs only when you invoke `fn()`.

---

# call() vs apply() vs bind()

| Method | Executes Immediately | Returns Function | Arguments |
|----------|----------------------|------------------|-----------|
| `call()` | ✅ Yes | ❌ No | Individual |
| `apply()` | ✅ Yes | ❌ No | Array |
| `bind()` | ❌ No | ✅ Yes | Individual |

---

# Real-World Example

```javascript
button.onclick = greet.bind(user);
```

`bind()` returns a function that will execute later when the button is clicked, while preserving the correct value of `this`.

If we used

```javascript
button.onclick = greet.call(user);
```

the function would execute immediately during assignment, which is not what we want.

# Interview Questions

## What is `bind()`?

`bind()` creates and returns a new function with the value of `this` permanently set.

---

## Does `bind()` execute the function immediately?

No.

It only returns a new function.

---

## What is the syntax of `bind()`?

```javascript
const fn = functionName.bind(thisArg, arg1, arg2);
```

---

## How are arguments passed in `bind()`?

Arguments are passed individually.

```javascript
const fn = greet.bind(user, "Delhi");
```

---

## What is the difference between `call()`, `apply()`, and `bind()`?

### call()

```javascript
greet.call(user, "Delhi");
```

- Executes immediately
- Arguments passed individually

---

### apply()

```javascript
greet.apply(user, ["Delhi"]);
```

- Executes immediately
- Arguments passed as an array

---

### bind()

```javascript
const fn = greet.bind(user, "Delhi");

fn();
```

- Does **not** execute immediately
- Returns a new function
- Arguments passed individually

---

## When is `bind()` commonly used?

When a function needs to be executed later while preserving the correct value of `this`.

Examples:

- Event Handlers
- Timers
- Callbacks
