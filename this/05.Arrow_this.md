# 📘 JavaScript Arrow Function & `this`

---

# Golden Rule ⭐⭐⭐⭐⭐

Arrow functions **do not have their own `this`.**

They inherit `this` from their **lexical (parent) scope**.

---

# Normal Function vs Arrow Function

## Normal Function

```javascript
function greet() {
    console.log(this);
}
```

Ask:

```text
Who called the function?
```

The caller decides `this`.

---

## Arrow Function

```javascript
const greet = () => {
    console.log(this);
};
```

Ask:

```text
Where was the arrow function created?
```

It inherits `this` from its parent (lexical) scope.

---

# Example 1

```javascript
const obj = {
    name: "Ashu",

    normal() {
        console.log(this.name);
    }
};

obj.normal();
```

Output

```text
Ashu
```

### Why?

```text
Caller = obj

↓

this = obj
```

---

# Example 2

```javascript
const obj = {
    name: "Ashu",

    arrow: () => {
        console.log(this.name);
    }
};

obj.arrow();
```

Output (Browser)

```text
undefined
```

### Why?

The arrow function is **not created inside a function**.

It is created in the **Global Execution Context**.

At the Global Execution Context (Browser)

```text
this = window
```

So the arrow captures

```text
this = window
```

Internally

```javascript
console.log(window.name);
```

Output

```text
undefined
```

---

# Example 3

```javascript
const obj = {

    name: "Ashu",

    normal() {

        const arrow = () => {
            console.log(this.name);
        };

        arrow();

    }

};

obj.normal();
```

Output

```text
Ashu
```

### Why?

The arrow is created inside `normal()`.

`normal()` is called by

```javascript
obj.normal();
```

So

```text
this = obj
```

The arrow inherits that `this`.

---

# Example 4

```javascript
const obj = {

    name: "Ashu",

    normal() {

        function inner() {
            console.log(this.name);
        }

        inner();

    }

};

obj.normal();
```

Output (Browser)

```text
undefined
```

### Why?

`inner()` is a **normal function**.

It has its own `this`.

It is called as

```javascript
inner();
```

No caller object.

So

```text
this = window
```

Internally

```javascript
console.log(window.name);
```

Output

```text
undefined
```

---

# Objects Do NOT Create Lexical Scope

```javascript
const obj = {
    arrow: () => {}
};
```

The object does **not** create a lexical scope.

The arrow is still created in the **Global Execution Context**.

Therefore it captures

```text
this = window
```

---

# Functions Create Lexical Scope

```javascript
function outer() {

    const arrow = () => {
        console.log(this);
    };

}
```

The arrow is created inside `outer()`.

It inherits

```text
outer()'s this
```

---

# Memory Trick

## Normal Function

```text
Who called me?

↓

That object becomes this.
```

---

## Arrow Function

```text
Where was I created?

↓

I inherit my parent's this.
```

---

# Interview Questions

## Does an Arrow Function have its own `this`?

No.

It inherits `this` from its lexical scope.

---

## How is `this` decided in a Normal Function?

By **how the function is called**.

---

## How is `this` decided in an Arrow Function?

By **where the function is created**.

---

## Do Objects Create Lexical Scope?

No.

Only functions (and the Global/Module Execution Context) create lexical scope.

---

## Why does this print `undefined`?

```javascript
const obj = {
    name: "Ashu",

    arrow: () => {
        console.log(this.name);
    }
};

obj.arrow();
```

Because the arrow function was created in the **Global Execution Context**, where `this` is `window` (browser).

---

# Normal Function vs Arrow Function

| Normal Function | Arrow Function |
|-----------------|----------------|
| Has its own `this` | Does not have its own `this` |
| `this` depends on the caller | `this` depends on the lexical scope |
| Ask **Who called me?** | Ask **Where was I created?** |
